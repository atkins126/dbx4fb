4.1.2007.18 / 4.1.2009.18 Release on 10 Sep 2009

  1. Return TWideMemoField if a blob data type declared as UTF8 character set. e.g.: BLOB SUB_TYPE 1 CHARACTER SET UTF8
  2. Fixed potential memory leak when TDBXDynalinkCommand.CreateParameterRow is invoked.  Previously, it require modification of DBX framework to call DBX_Close after consuming ParameterRow.  This fix doesn't require modification of DBX framework but will keep a list of ParameterRow in TDBXCommand_Firebird instance and all ParameterRow references will set to nil after TDBXCommand_Firebird instance has closed.  See QC#57849  for more details.

4.1.2007.14 / 4.1.2009.14 Release on 07 Jul 2009

  1. Add: support Firebird Stored Procedure and DBX TSQLStoredProc
  2. Update: In Delphi 2007 DBX 4 driver, BIGINT columns map to TFmtBcdField.  While perform data coering operation, Numeric and Decimal columns map to SQL_INT64 data type in some situation map to TDBXDataTypes.BcdType and TFmtBcdField of VCL persistent field.  The above behavior has changed for DBX 4 unicode driver used by Delphi 2009, BIGINT column map to TDBXTypes.Int64Type.  It convert to TDBXTypes.BcdType previously.  As such, it map to TLargeIntField in VCL's persistent field.
